记账小系统 — 架构方案对比

纯前端、localStorage 存储、桌面 Web 端

A

单页全功能

所有功能在一页上：顶部是记账表单，下面是交易列表，侧边有统计摘要。无需切换页面，一目了然。

优点

  • 所见即所得，无需导航
  • 开发最简单
  • 适合少量功能

缺点

  • 功能多了会拥挤
  • 统计区域空间有限
B

双页标签式（推荐）

"记账"和"统计"两个 Tab 页切换。记账页专注录入和查看流水，统计页用图表展示分类汇总。

优点

  • 职责清晰，各页面不拥挤
  • 统计可以做得更丰富
  • 后续加功能也方便

缺点

  • 需要在两个页面间切换
  • 稍微多一点开发量
C

电子表格风格

像一个精简的 Excel，所有记录按行排列，字段以列展示，可以直接在表格中编辑。统计用表格透视或简单图表。

优点

  • 数据密度高，批量操作方便
  • 适合习惯表格的用户

缺点

  • 录入体验不如表单
  • 移动端几乎不可用
  • 视觉上不够温暖
